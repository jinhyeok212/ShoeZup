## Git Rules

- main 브랜치 직접 push 금지
- 기능별 브랜치 생성 후 작업 (기능별 브랜치에 작업 후 최종 안정 코드를 main에 PR - 코드 안정화 및 충돌 방지 목적)
- 브랜치 생성 -> 작업 -> Pull Request -> 리뷰 Merge 순으로 진행
- Pull Request 생성 후 최소 1명 리뷰 후 merge 

## Branch Naming (기능별 브랜치 생성 시 브랜치 이름)

- feature/기능명
- fix/수정내용
- docs/문서내용

## Commit Convention (커밋할 때 기입할 내용)

- feat: 기능 추가
- fix: 버그 수정
- docs: 문서 수정
- refactor: 코드 구조 개선
- chore: 설정 변경
